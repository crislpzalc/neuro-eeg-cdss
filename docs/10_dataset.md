# Dataset

## 1. Data Source

This project uses the **CHB-MIT Scalp EEG Database** in **BIDS format**, obtained from the BIDS-converted release available through Zenodo. The dataset consists of scalp EEG recordings annotated for seizure events and is widely used in seizure detection research.

The BIDS organization provides a standardized and reproducible structure for signal files and metadata, facilitating automated data access and downstream processing.

---

## 2. Dataset Characteristics

The dataset used in this project contains:

* **23 subjects**
* **Sampling frequency:** 256 Hz
* **18 EEG channels**
* **Bipolar montage ("double banana")**
* **Recordings of approximately one hour**
* **Event annotations** stored in `events.tsv`

Each EEG recording is associated with metadata and temporal annotations describing seizure and background activity.

---

## 3. Data Organization

The dataset is stored locally following the BIDS directory structure, with raw data separated from processed artifacts.

At the project level, the data organization is divided into:

* `data/raw/` for original downloaded data
* `data/manifests/` for indexed metadata
* `data/processed/` for derived datasets generated by the pipeline

A manifest file was created to provide a centralized index of all EEG recordings:

```text
data/manifests/manifest.parquet
```

The manifest stores structured information required to access each recording programmatically, including subject identity and file path.

---

## 4. Annotation Format

Seizure annotations are provided through BIDS `events.tsv` files. Each file contains temporal events defined relative to the start of the recording.

The key fields used in this project are:

* `onset`: start time of the event in seconds
* `duration`: duration of the event in seconds
* `eventType`: event category

According to the dataset metadata (`events.json`):

* `bckg` denotes background activity
* event types beginning with `sz` denote seizure activity

This project uses the `eventType` field as the authoritative source for identifying seizure intervals.

---

## 5. Dataset Preparation for Supervised Learning

Raw EEG recordings cannot be directly used as input for baseline classical machine learning models. Therefore, the recordings were transformed into a structured tabular dataset composed of fixed-length temporal segments.

The preparation pipeline includes four main steps:

1. **Event parsing**
2. **Temporal segmentation**
3. **Overlap computation**
4. **Label assignment**

---

## 6. Event Parsing

For each EEG recording, seizure intervals are extracted from the corresponding `events.tsv` file.

A temporal interval is considered a seizure interval when its `eventType` starts with `sz`. For every seizure event, the following values are derived:

* `onset_sec`
* `duration_sec`
* `end_sec = onset_sec + duration_sec`

This results in a recording-level set of seizure intervals that can later be compared against fixed time windows.

---

## 7. Temporal Segmentation

Each recording is segmented into **fixed-length windows of 5 seconds**.

Only complete windows are retained. In other words, a segment is included only if its end time remains within the recording duration.

The segmentation strategy uses:

* **Window size:** 5 seconds
* **Stride:** 5 seconds

This produces **non-overlapping windows**.

### Rationale

The choice of 5-second windows aims to balance two competing requirements:

* preserving sufficient temporal resolution to localize seizure activity
* maintaining enough signal duration to compute stable statistical and spectral features

The use of non-overlapping windows provides a simple and reproducible baseline while reducing redundancy between adjacent samples.

---

## 8. Overlap Computation

Each window is compared against the seizure intervals of the corresponding recording.

For every segment, a seizure overlap duration is computed as the total time shared between the window and any seizure interval. From this value, an **overlap ratio** is defined:

```text
overlap_ratio = seizure_overlap_duration / window_duration
```

Because window duration is fixed at 5 seconds, the overlap ratio takes values between 0 and 1.

This overlap ratio is the central quantity used for label assignment.

---

## 9. Label Assignment

Each temporal segment is assigned a binary label according to its overlap with seizure intervals.

The labeling strategy is defined as follows:

* **label = 1** if `overlap_ratio >= 0.5`
* **label = 0** if `overlap_ratio = 0`
* windows with `0 < overlap_ratio < 0.5` are discarded

### Rationale

This strategy was chosen to reduce label noise near seizure boundaries.

In seizure detection, windows located at the start or end of a seizure may contain a mixture of ictal and non-ictal activity. Assigning those segments directly to either class introduces ambiguity. By discarding partially overlapping windows below the threshold, the dataset retains cleaner positive and negative examples for the baseline phase.

This decision is especially important given the clinical sensitivity of false negatives and the severe class imbalance of the task.

---

## 10. Output Dataset

The final processed dataset is stored as:

```text
data/processed/segments.parquet
```

Each row corresponds to a labeled temporal segment. The dataset includes the following fields:

* `subject`
* `session`
* `run`
* `path`
* `recording_duration_sec`
* `start_sec`
* `end_sec`
* `window_size_sec`
* `stride_sec`
* `overlap_ratio`
* `label`

This format provides a direct interface between raw data and downstream feature extraction or modeling.

---

## 11. Dataset Statistics

After preprocessing and labeling, the resulting dataset contains:

* **Total segments:** 707,524
* **Negative segments:** 705,203
* **Positive segments:** 2,321

This corresponds to approximately:

* **99.7% non-seizure**
* **0.3% seizure**

The resulting dataset is therefore **extremely imbalanced**, which is expected in real seizure detection settings where seizure events are sparse relative to long background recordings.

---

## 12. Validation and Quality Control

Several sanity checks were performed on the generated dataset:

* no missing values in critical fields
* all segment durations equal to 5 seconds
* all overlap ratios within the range [0, 1]
* binary labels restricted to {0, 1}
* all 23 subjects present in the final dataset

These checks confirm the internal consistency of the preprocessing pipeline and the integrity of the final segment-level dataset.

---

## 13. Limitations

The current dataset preparation stage has several limitations that should be acknowledged.

First, the segment-level representation assumes temporal independence between windows, which is not strictly true for EEG signals.

Second, the labeling strategy depends directly on the quality and temporal precision of the original annotations.

Third, no signal-level preprocessing, artifact handling, or normalization has yet been applied at this stage.

Finally, the generated dataset remains highly imbalanced, which will directly affect model training and evaluation.

---

## 14. Role in the Overall Pipeline

This dataset constitutes the first supervised learning-ready representation of the raw EEG recordings.

It serves as the foundation for the next stages of the project:

* feature extraction
* patient-independent model training
* clinical evaluation
* future deep learning extensions

Because all subsequent stages depend on the correctness of this transformation, dataset construction is treated as a critical and fully reproducible component of the pipeline.
